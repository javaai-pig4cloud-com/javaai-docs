# 提示工程模式

本指南涵盖了 Spring AI 应用程序中提示工程的常见模式和最佳实践。

## 概述

提示工程是设计和优化提示以获得 AI 模型最佳结果的实践。

## 常见模式

### 思维链

```java
String prompt = """
    让我们一步步解决这个问题：
    1. 首先，分析问题
    2. 然后，将其分解为部分
    3. 最后，提供解决方案
    
    问题：{problem}
    """;
```

### 少样本学习

```java
String prompt = """
    这里有一些例子：
    
    示例 1：
    输入："法国的首都是什么？"
    输出："巴黎"
    
    示例 2：
    输入："日本的首都是什么？"
    输出："东京"
    
    现在，回答这个问题：
    输入："{question}"
    """;
```

### 基于角色的提示

```java
String prompt = """
    你是一位专业的软件架构师。
    你的任务是审查以下代码并提供建议：
    
    {code}
    """;
```

## 最佳实践

1. 具体且清晰
2. 提供上下文
3. 尽可能使用示例
4. 分解复杂任务
5. 包含约束和要求

## 高级技术

### 温度控制

```java
ChatOptions options = ChatOptions.builder()
    .withTemperature(0.7)
    .build();
```

### 系统消息

```java
String systemMessage = """
    你是一位专注于 Java 开发的 AI 助手。
    始终提供 Java 代码示例。
    """;
```

## 测试和评估

- 使用测试用例验证提示
- 衡量响应质量
- 根据结果迭代和改进
