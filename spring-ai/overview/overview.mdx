`Spring AI` 项目旨在简化应用程序开发过程中集成人工智能功能的复杂性。

该项目从著名的Python项目如LangChain和LlamaIndex中汲取灵感，但Spring AI并非这些项目的直接移植。
项目创立基于这样的信念：生成式AI应用的下一波浪潮不会仅限于Python开发者，而将在众多编程语言中普及。

<Note>Spring AI解决了AI集成的基本挑战：**将企业数据和API与AI模型连接起来**。</Note>

<Frame caption="Spring AI 集成">
  <img src="/images/spring-ai-integration-diagram-3.svg" width="500" />
</Frame>

Spring AI提供了作为开发AI应用程序基础的抽象层。
这些抽象拥有多种实现方式，使得在最少代码更改的情况下轻松切换组件成为可能。

## 特性

Spring AI提供以下特性：

<AccordionGroup>
  <Accordion title="模型支持" icon="robot">
    为聊天、文本到图像和嵌入模型提供跨AI提供商的可移植API支持。同时支持同步和流式API选项。也可以访问模型特定的功能。
  </Accordion>
  
  <Accordion title="AI模型提供商" icon="building">
    支持所有主要的AI模型提供商，如Anthropic、OpenAI、Microsoft、Amazon、Google和Ollama。支持的模型类型包括：
    
    - [聊天补全](/spring4ai/api/chatmodel)
    - [嵌入](/spring4ai/api/embeddings)
    - [文本生成图像](/spring4ai/api/imageclient)
    - [音频转录](/spring4ai/api/audio/transcriptions)
    - [文本转语音](/spring4ai/api/audio/speech)
    - [内容审核](/spring4ai/api/moderation)
  </Accordion>
  
  <Accordion title="结构化输出" icon="code">
    [结构化输出](/spring4ai/api/structured-output-converter) - 将AI模型输出映射到POJO对象。
  </Accordion>
  
  <Accordion title="向量数据库支持" icon="database">
    支持所有主要的向量数据库提供商，如Apache Cassandra、Azure Cosmos DB、Azure Vector Search、Chroma、Elasticsearch、GemFire、MariaDB、Milvus、MongoDB Atlas、Neo4j、OpenSearch、Oracle、PostgreSQL/PGVector、PineCone、Qdrant、Redis、SAP Hana、Typesense和Weaviate。
  </Accordion>
  
  <Accordion title="工具与函数调用" icon="wrench">
    [工具/函数调用](/spring4ai/api/tools) - 允许模型请求执行客户端工具和函数，从而在需要时访问必要的实时信息并采取行动。
  </Accordion>
  
  <Accordion title="可观测性" icon="eye">
    [可观测性](/spring4ai/observability) - 提供AI相关操作的洞察。
  </Accordion>
  
  <Accordion title="ETL框架" icon="gears">
    用于数据工程的文档摄取[ETL框架](/spring4ai/api/etl-pipeline)。
  </Accordion>
  
  <Accordion title="模型评估" icon="chart-line">
    [AI模型评估](/spring4ai/api/testing) - 帮助评估生成内容并防止幻觉响应的工具。
  </Accordion>
  
  <Accordion title="Spring Boot集成" icon="spring">
    为AI模型和向量存储提供Spring Boot自动配置和启动器。
  </Accordion>
  
  <Accordion title="ChatClient API" icon="comments">
    [ChatClient API](/spring4ai/api/chatclient) - 用于与AI聊天模型通信的流畅API，风格类似于WebClient和RestClient API。
  </Accordion>
  
  <Accordion title="Advisors API" icon="user-gear">
    [Advisors API](/spring4ai/api/advisors) - 封装了常见的生成式AI模式，转换发送到语言模型(LLM)和从语言模型接收的数据，并提供跨各种模型和用例的可移植性。
  </Accordion>
  
  <Accordion title="记忆与RAG" icon="brain">
    支持[聊天对话记忆](/spring4ai/api/chatclient#chat-memory)和[检索增强生成(RAG)](/spring4ai/api/chatclient#retrieval-augmented-generation)。
  </Accordion>
</AccordionGroup>

这些特性集使您能够实现常见用例，例如"基于文档的问答"或"与文档对话"。
